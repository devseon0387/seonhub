Dev Workspace — 서비스 메타데이터 시안

각 프로젝트가 어떤 배포·DB·VCS 서비스를 쓰는지 카드에서 바로 보이게 하는 4가지 방향. 샘플: Plick (로컬 개발) · 메뉴한컷 (Vercel 배포) · SEON Hub (자체 SQLite + GitHub).

Variant A

Status pill + 서비스 뱃지 행

기존 기술 스택 뱃지 아래에 "서비스 뱃지" 한 줄을 추가. 배포된 곳은 초록색 강조(prod), 나머지는 중립 회색. 가장 적은 공간 차지. 기존 카드 구조 거의 그대로 유지.

✓ 정보 밀도 우수 ✓ 기존 카드 수정 최소 ✗ URL 직접 노출 X (클릭으로 해결)

Variant B

서비스 로고 그리드 + 배포 URL

각 서비스를 브랜드 컬러의 로고 박스로 표시 (V=Vercel, S=Supabase, G=GitHub). 라이브 서비스엔 초록 상태 점. 배포된 카드엔 URL이 직접 노출되어 클릭 한 번에 접속.

✓ 시각적으로 가장 인식 빠름 (로고) ✓ 배포 URL 1클릭 ✗ 공간 차지 많음
