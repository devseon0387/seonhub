새 프로젝트 위저드 리디자인

현재 6스텝(시작→유형→클라이언트→프로젝트→회차→완료). 파트너/매니저 제거된 상태에서 과한 구조. 타입별 메타 필드 입력 기회도 없음.

현재 위저드의 문제

  • "시작 방법" 스텝(클라이언트부터/프로젝트만)은 실질 가치 낮음 — 클라 선택은 프로젝트 폼에 통합 가능
  • 6스텝은 과함 — 대부분 1~2개 필드만 입력
  • 카테고리 옵션(유튜브/브이로그/기업홍보/교육)이 영상 전용인데 dev/content 타입에도 강제 노출
  • 타입별 메타 필드(개발 리포, 콘텐츠 플랫폼) 입력 기회 없음 — 생성 후 편집 UI도 없음
  • 하위 단위 자동 생성(회차/스프린트/발행)은 대부분의 경우 필요 없고 혼란 유발

시안 (5개)

  • A안 — 3스텝 컴팩트 (유형 → 기본정보+메타 → 완료)
  • B안 — 2스텝 초간단 (유형 → 올인원 폼)
  • C안 — 타입 탭 + 단일 폼 (스텝 없음)
  • D안 — 스마트 입력 (제목만 → 고급 옵션 접기) ⭐ 추천
  • E안 — 현 구조 정리 (4스텝, 시작 제거 + 메타 추가)
